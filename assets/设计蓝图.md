
# AI 学习平台 — 设计蓝图

> 版本：v1.3（通用版） | 定位：面向大众的本地视频学习平台模板

---

## 一、定位澄清

| 传统做法 | 本方案 |
|----------|--------|
| 全栈 Web（前端+后端+数据库+部署） | 本地 HTTP 服务 + 单 HTML，启动即用 |
| 先建平台再填充内容 | 课程路径映射优先，功能和内容同步迭代 |
| 多用户、注册登录 | 单人使用，数据存 localStorage |
| MySQL / PostgreSQL | 零数据库依赖 |

一句话：一个本地运行的课程操作系统，不是内容管理后台。

---

## 二、核心功能矩阵（四层架构）

```
┌──────────────────────────────────────────────────────┐
│                  第四层：反馈回路                        │
│  学习日志 → 周度报告 → 弱项可视化 → 下一优先推荐         │
├──────────────────────────────────────────────────────┤
│                  第三层：学练测闭环                       │
│  标注文本 → AI 自动出题 → 做题 → 错题回链到原资料         │
├──────────────────────────────────────────────────────┤
│                  第二层：学习引擎                         │
│  交叉学习路径 → 章节完成勾选 → 进度仪表盘 → 每日推荐       │
├──────────────────────────────────────────────────────┤
│                  第一层：知识入口                         │
│  课程映射 → 目录树导航 → 本地文件一键打开                  │
└──────────────────────────────────────────────────────┘
```

开发顺序就是 一 → 二 → 三 → 四。每一层依赖上一层的数据。

---

## 三、技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 运行环境 | 本地 HTTP 服务 + 浏览器 | Python 启动 HTTP 服务，浏览器访问 localhost |
| 样式 | 原生 CSS（暗色/亮色主题） | 不引入框架，最小依赖 |
| 逻辑 | 原生 JavaScript（ES6+） | 不用 React/Vue，降低心智负担 |
| 数据存储 | localStorage | 进度/标注/做题记录本地持久化 |
| 课程配置 | 独立 JSON 文件，fetch 加载 | 课程变更只改 JSON，不动代码 |
| AI 总结 | DeepSeek API（可选） | 不设置 API Key 也不影响核心功能 |

禁止引入的依赖：Node.js、npm、Webpack、React、Vue、任何需要 `npm install` 的东西。

## 四、数据模型

### 4.1 课程配置 JSON（course-data.json）

```json
[
  {
    "id": "stage_01_basics",
    "name": "阶段一 基础入门",
    "lessons": [
      {
        "id": "lesson_01_intro",
        "name": "课程介绍与环境搭建",
        "source": "基础教程",
        "video": [
          "/video/course-a/01_课程介绍.mp4",
          "/video/course-a/02_环境搭建.mp4"
        ],
        "notes": "本节课的核心要点..."
      }
    ]
  }
]
```

字段说明：

| 字段 | 说明 |
|------|------|
| `id` | 唯一标识（英文+数字+下划线） |
| `name` | 显示名称 |
| `source` | 课程来源标签，决定界面颜色 |
| `video` | 视频 URL 数组（自动组成播放列表） |
| `notes` | 可选，Markdown 格式的笔记内容 |

视频 URL 格式：`/video/{prefix}/{relative-path}`，其中 `{prefix}` 对应 server.py 中 `VIDEO_ROOTS` 的 key。

### 4.2 用户进度数据（localStorage key: `learn_state`）

```json
{
  "completed": ["lesson_id_1", ...],
  "starred": ["lesson_id_2", ...],
  "annotations": { "lesson_id": { "marks": [...], "completedDate": "YYYY-MM-DD" } },
  "playPositions": { "lesson_id": 120.5 },
  "watchTime": { "YYYY-MM-DD": 3600 },
  "watchHistory": [ { "lessonId", "lessonName", "stageName", "timestamp", "progress" } ],
  "goal": 5, "goalMode": "lessons", "goalMinutes": 60,
  "theme": "dark", "speed": 1
}
```

设计原则：
- 所有 lesson_id 与 course-data.json 一一对应，跨课程唯一
- 不做版本迁移，JSON 变了就重建数据结构

## 六、关键设计决策

### 为什么不做数据库？

单人使用，数据量不超过 1MB。localStorage 的 5MB 上限绰绰有余。引入数据库是过度工程。

### 为什么不用框架？

项目核心价值在内容和路径设计，不在技术栈。原生 JS 在可控代码量内能写完所有交互。引入框架会多出构建工具链、状态管理、组件拆分，每一项都是阻力。

### 为什么不先做 AI 功能？

AI 功能依赖 API Key、Python 环境、Prompt 调优。在导航和进度都没跑通之前做 AI，等于在一个不能用的系统上优化。先让系统追踪"你学到了哪"，再解决"你学会了没"。

### 浏览器 file:// 安全限制

当 HTML 通过 file:// 协议打开时，浏览器的安全沙箱会阻止跨盘符加载本地视频。解决方案：启动 Python HTTP 服务，浏览器通过 http://localhost:8765 访问，本地 HTTP 服务可以访问任意盘符的文件。

## 七、分阶段开发路线图

### Phase 1：课程导航（核心交付物）

目标：启动 server.py，看到完整课程树，点击能播放视频。

功能清单：
- 加载 course-data.json，渲染左侧目录树
- 阶段折叠/展开
- 点击章节 → 加载视频播放器 + 播放列表
- 章节完成状态勾选（存 localStorage）
- 播放列表自动续播
- 断点续播（存播放位置）

### Phase 2：进度仪表盘

目标：可视化学习进度，不再靠感觉。

功能清单：
- 今日完成数 / 总数统计卡片
- 各阶段完成百分比进度条
- 连续学习天数
- 每日学习时长记录
- 收藏课程管理
- 观看历史记录

### Phase 3：AI 智能总结

目标：视频内容自动总结 + 出题。

功能清单：
- AI 总结面板（DeepSeek API）
- Whisper 语音转录（可选）
- 自动出题（填空/判断/简答）
- 总结缓存到本地文件

### Phase 4：反馈回路

目标：不只是学，还能看见自己哪里薄弱。

功能清单：
- 弱项可视化
- 周度学习报告
- 遗忘曲线间隔提醒
- 复习推荐权重算法

## 八、快速上手

第一步：创建平台运行目录，复制 learn.html / server.py / course-data.json 到该目录（视频文件留在原位，不放入此目录）
第二步：在 server.py 中配置 VIDEO_ROOTS（填入你的视频实际存放目录）
第三步：编辑 course-data.json 添加你的课程（按上面 4.1 的格式）
第四步：运行 `python server.py`，浏览器自动打开
第五步（可选）：设置 `DEEPSEEK_API_KEY` 环境变量启用 AI 总结

详见 SKILL.md 中的 Getting Started 部分。
